🧠 DSA — MCQ Questions (30+)
Q1. What is the time complexity of binary search? HOT
a) O(n)   b) O(n²)   c) O(log n)   d) O(n log n)

✅ Answer: c) O(log n)

Binary search halves the search space each step → log₂(n) comparisons.

Q2. Which data structure uses FIFO ordering?
a) Stack   b) Queue   c) Tree   d) Graph

✅ Answer: b) Queue

Queue = First In First Out. Stack = Last In First Out (LIFO).

Q3. Which data structure is used for BFS traversal? HOT
a) Stack   b) Queue   c) Heap   d) Linked List

✅ Answer: b) Queue

BFS uses a Queue. DFS uses a Stack (or recursion).

Q4. What causes a stack overflow?

✅ Answer: Infinite recursion or very deep recursion exceeding call stack size.

Each recursive call adds a frame to the call stack. If there's no base case or recursion is too deep, the stack memory is exhausted.

Q5. Which sorting algorithm is stable? HOT
a) Quick Sort   b) Heap Sort   c) Merge Sort   d) Selection Sort

✅ Answer: c) Merge Sort

Stable sort = equal elements maintain their original order. Merge Sort is stable. Quick Sort and Heap Sort are NOT stable.

Q6. Best case time complexity of Quick Sort?
a) O(n)   b) O(n log n)   c) O(n²)   d) O(log n)

✅ Answer: b) O(n log n)

Best/Average: O(n log n). Worst case (already sorted, bad pivot): O(n²).

Q7. Average time complexity of HashMap get/put?
a) O(n)   b) O(log n)   c) O(1)   d) O(n log n)

✅ Answer: c) O(1) — amortized

HashMap uses hashing for O(1) average. Worst case O(n) due to collisions.

Q8. What is amortized time complexity?

✅ The average time per operation over a worst-case sequence of operations.

Example: ArrayList/dynamic array doubling — single resize is O(n), but averaged over n insertions it's O(1) per insert.

Q9. Difference between Array and Linked List? HOT
  • Array: Contiguous memory, O(1) access by index, O(n) insert/delete
  • Linked List: Non-contiguous, O(n) access, O(1) insert/delete at head
  • Array has better cache performance. LL has dynamic size.
Q10. What is the height of a balanced BST with n nodes?

✅ O(log n)

A balanced BST ensures left and right subtrees differ in height by at most 1 → height is log₂(n).

Q11. Inorder traversal of a BST gives?

✅ Sorted (ascending) order

Inorder: Left → Root → Right. For BST, this always produces sorted output.

Q12. Which data structure is best for implementing an undo feature?

✅ Stack

Each action is pushed. Undo = pop the last action. LIFO perfectly models undo.

Q13. What is the worst-case time for searching in an unsorted array?

✅ O(n) — linear search

Q14. Difference between DFS and BFS? HOT
  • DFS: Goes deep first. Uses Stack/recursion. Less memory for narrow graphs.
  • BFS: Goes level by level. Uses Queue. Finds shortest path in unweighted graphs.
Q15. What is a collision in hashing? How to handle it?

✅ When two different keys produce the same hash index.

  • Chaining: Store multiple elements at same index using linked list
  • Open Addressing: Find next empty slot (linear probing, quadratic probing)
Q16. Difference between Stack and Queue?
  • Stack: LIFO — push/pop from top only
  • Queue: FIFO — enqueue at rear, dequeue from front
Q17. What is a Priority Queue?

✅ A queue where elements are dequeued based on priority, not insertion order.

Usually implemented using a Heap. Min-Heap: smallest priority first. Max-Heap: largest first.

Q18. Time complexity of inserting at the beginning of an ArrayList vs LinkedList?
  • ArrayList: O(n) — must shift all elements
  • LinkedList: O(1) — just change head pointer
Q19. What is a complete binary tree?

✅ All levels fully filled except possibly the last, which is filled left to right.

A heap is always a complete binary tree.

Q20. What is the space complexity of merge sort?

✅ O(n)

Merge sort needs extra space for temporary arrays during merging.

C1. Two Sum (LeetCode #1) CONFIRMED

Given array nums and target, return indices of two numbers that add up to target.

function twoSum(nums, target) {
  const map = new Map();
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (map.has(complement)) return [map.get(complement), i];
    map.set(nums[i], i);
  }
}
// Time: O(n), Space: O(n)
C2. Valid Parentheses (LeetCode #20) CONFIRMED TENNR

Check if bracket string is valid. Tennr adds variations with <> and || brackets.

function isValid(s) {
  const stack = [];
  const map = { ')':'(', '}':'{', ']':'[' };
  for (const c of s) {
    if ('({['.includes(c)) stack.push(c);
    else if (stack.pop() !== map[c]) return false;
  }
  return stack.length === 0;
}
// Time: O(n), Space: O(n)
C3. Top K Frequent Elements (LeetCode #347) CONFIRMED TENNR

Return the k most frequent elements from an array.

function topKFrequent(nums, k) {
  const freq = {};
  for (const n of nums) freq[n] = (freq[n] || 0) + 1;
  // Bucket sort approach
  const buckets = Array.from({length: nums.length + 1}, () => []);
  for (const [num, count] of Object.entries(freq)) {
    buckets[count].push(Number(num));
  }
  const result = [];
  for (let i = buckets.length - 1; i >= 0 && result.length < k; i--) {
    result.push(...buckets[i]);
  }
  return result.slice(0, k);
}
// Time: O(n), Space: O(n)
C4. Maximum Subarray — Kadane's (LeetCode #53) HOT
function maxSubArray(nums) {
  let maxSoFar = nums[0], maxEndingHere = nums[0];
  for (let i = 1; i < nums.length; i++) {
    maxEndingHere = Math.max(nums[i], maxEndingHere + nums[i]);
    maxSoFar = Math.max(maxSoFar, maxEndingHere);
  }
  return maxSoFar;
}
// Time: O(n), Space: O(1)
C5. Reverse a Linked List (LeetCode #206) HOT
function reverseList(head) {
  let prev = null, curr = head;
  while (curr) {
    const next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
  }
  return prev;
}
// Time: O(n), Space: O(1)
C6. Best Time to Buy and Sell Stock (LeetCode #121)
function maxProfit(prices) {
  let minPrice = Infinity, maxProfit = 0;
  for (const price of prices) {
    minPrice = Math.min(minPrice, price);
    maxProfit = Math.max(maxProfit, price - minPrice);
  }
  return maxProfit;
}
// Time: O(n), Space: O(1)
C7. Valid Anagram (LeetCode #242)
function isAnagram(s, t) {
  if (s.length !== t.length) return false;
  const count = {};
  for (const c of s) count[c] = (count[c] || 0) + 1;
  for (const c of t) {
    if (!count[c]) return false;
    count[c]--;
  }
  return true;
}
// Time: O(n), Space: O(1) — fixed 26 chars
C8. Binary Search (LeetCode #704)
function search(nums, target) {
  let l = 0, r = nums.length - 1;
  while (l <= r) {
    const mid = l + Math.floor((r - l) / 2);
    if (nums[mid] === target) return mid;
    else if (nums[mid] < target) l = mid + 1;
    else r = mid - 1;
  }
  return -1;
}
// Time: O(log n), Space: O(1)
C9. Detect Cycle in Linked List (LeetCode #141)
function hasCycle(head) {
  let slow = head, fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) return true;
  }
  return false;
}
// Floyd's Tortoise & Hare — O(n) time, O(1) space
C10. Group Anagrams (LeetCode #49) REPORTED
function groupAnagrams(strs) {
  const map = {};
  for (const s of strs) {
    const key = s.split('').sort().join('');
    if (!map[key]) map[key] = [];
    map[key].push(s);
  }
  return Object.values(map);
}
// Time: O(n * k log k), Space: O(n)
⚡ JavaScript — MCQ Questions (25+)
Q1. What is the difference between == and ===? HOT

✅ == does type coercion before comparison. === checks both value AND type (strict).

"5" == 5   // true (coerces string to number)
"5" === 5  // false (different types)
Q2. What is hoisting in JavaScript? HOT

✅ Variable and function declarations are moved to the top of their scope during compilation.

  • var: hoisted and initialized as undefined
  • let/const: hoisted but NOT initialized (Temporal Dead Zone — ReferenceError if accessed before declaration)
  • function declarations: fully hoisted (can call before declaration)
Q3. What is a closure? HOT

✅ A function that retains access to its outer (lexical) scope variables even after the outer function returns.

function outer() {
  let count = 0;
  return function inner() { return ++count; };
}
const fn = outer();
fn(); // 1
fn(); // 2  — count persists via closure
Q4. Difference between var, let, const? HOT
  • var: function-scoped, hoisted (undefined), can redeclare
  • let: block-scoped, hoisted (TDZ), can reassign, no redeclare
  • const: block-scoped, hoisted (TDZ), cannot reassign, must initialize
Q5. What is the event loop? HOT

✅ Mechanism that handles async operations in single-threaded JS.

Call Stack → Web APIs → Task Queue / Microtask Queue → Event Loop checks if stack is empty, then pushes next task. Microtasks (Promises) run before Macrotasks (setTimeout).

Q6. What is the output? setTimeout(() => console.log(1), 0); Promise.resolve().then(() => console.log(2)); console.log(3); HOT

✅ Output: 3, 2, 1

3 (sync) → 2 (microtask/Promise) → 1 (macrotask/setTimeout)

Q7. What is a Promise?

✅ An object representing the eventual completion (or failure) of an async operation.

States: Pending → Fulfilled / Rejected. Use .then() for success, .catch() for error, .finally() for cleanup.

Q8. Difference between null and undefined?
  • undefined: Variable declared but not assigned. JS engine sets it.
  • null: Intentional absence of value. Programmer sets it.
  • typeof undefined → "undefined". typeof null → "object" (JS bug).
Q9. What does Array.map() do vs forEach()?
  • map(): Returns a NEW array with transformed elements
  • forEach(): Executes function on each element, returns undefined
Q10. What is 'this' in JavaScript? IMP
  • Global context: window (browser) / global (Node)
  • Object method: the object calling the method
  • Arrow function: inherits from enclosing scope (lexical)
  • Constructor (new): the newly created object
  • call/apply/bind: explicitly set
Q11. What is event delegation?

✅ Attaching a single event listener to a parent element to handle events on child elements via event bubbling.

Benefits: fewer listeners, works for dynamically added elements.

Q12. What is the spread operator (...)?

✅ Expands an iterable into individual elements.

const arr = [1, 2, 3];
const copy = [...arr];       // shallow copy
const merged = [...arr, 4];  // [1,2,3,4]
const obj = {...{a:1}, b:2}; // {a:1, b:2}
Q13. What is destructuring?
const [a, b] = [1, 2];              // a=1, b=2
const {name, age} = {name:'X', age:20}; // name='X'
Q14. Difference between arrow function and regular function?
  • Arrow: no own 'this', no 'arguments' object, cannot be used as constructor
  • Regular: has own 'this', has 'arguments', can be constructor
Q15. What is callback hell? How to solve it?

✅ Deeply nested callbacks making code hard to read ("pyramid of doom").

Solutions: Promises → async/await → modularizing functions.
