🔷 OOP — Questions (20+)
Q1. What are the 4 pillars of OOP? HOT

✅ Encapsulation, Abstraction, Inheritance, Polymorphism

  • Encapsulation: Bundling data + methods; hiding internal state (private fields)
  • Abstraction: Showing only essential features; hiding implementation details
  • Inheritance: Child class inherits parent's properties/methods → code reuse
  • Polymorphism: Same interface, different implementations
Q2. Method Overriding vs Overloading? HOT
  • Overriding: Child redefines parent's method. Same name + same parameters. Runtime polymorphism.
  • Overloading: Same name, different parameters (number/type). Compile-time polymorphism.
Q3. Abstract class vs Interface? HOT
  • Abstract class: Can have both abstract and concrete methods. Single inheritance. Can have constructors and instance variables.
  • Interface: All methods abstract (before Java 8). Multiple inheritance allowed. No constructors. Fields are public static final.
Q4. What is encapsulation? Give a real-world example.

✅ Wrapping data and methods into a single unit (class) and controlling access.

Example: A bank account class — balance is private. You can only access/modify it through deposit() and withdraw() methods. You can't directly change the balance variable.

Q5. What is polymorphism? Types?

✅ Same method name behaves differently based on the object/context.

  • Compile-time: Method overloading, operator overloading
  • Runtime: Method overriding (virtual functions). Decided at runtime.
Q6. What is the SOLID principle? IMP
  • S — Single Responsibility: One class = one job
  • O — Open/Closed: Open for extension, closed for modification
  • L — Liskov Substitution: Subtypes must be substitutable for their base types
  • I — Interface Segregation: Many specific interfaces > one fat interface
  • D — Dependency Inversion: Depend on abstractions, not concretions
Q7. What is a constructor? Can it be private?

✅ Special method called when object is created. Same name as class, no return type.

Yes, constructor can be private — used in Singleton pattern to restrict object creation.

Q8. What is the Singleton pattern? IMP

✅ Ensures only ONE instance of a class exists globally.

How: Private constructor + static method that returns the single instance. Example: Database connection pool, Logger.

Q9. What is the Factory pattern?

✅ Creates objects without specifying the exact class. A factory method decides which class to instantiate based on input.

Example: ShapeFactory.create("circle") returns a Circle object.

Q10. What is MVC? HOT
  • Model: Data + business logic
  • View: UI / presentation layer
  • Controller: Handles user input, updates Model and View

Separates concerns → easier to maintain and test.

Q11. What is the static keyword?

✅ Static members belong to the CLASS, not any specific object. Shared across all instances. Accessed via ClassName.method().

Q12. What is multiple inheritance? Why is it a problem?

✅ A class inheriting from more than one parent. Problem: Diamond Problem — ambiguity when two parents have the same method.

Java solves this by NOT allowing multiple class inheritance but allowing multiple interface implementation.

Q13. Access modifiers: public, private, protected?
  • public: Accessible everywhere
  • private: Accessible only within the same class
  • protected: Accessible within same class + subclasses
  • default (Java): Accessible within same package
Q14. What is coupling and cohesion?
  • Coupling: How dependent two modules are. LOW coupling = good (modules are independent).
  • Cohesion: How related the responsibilities within a module are. HIGH cohesion = good (module does one thing well).
🗄️ SQL — Questions (20+)
Q1. What is the difference between WHERE and HAVING? HOT
  • WHERE: Filters ROWS before grouping. Cannot use aggregate functions.
  • HAVING: Filters GROUPS after GROUP BY. Can use aggregate functions.
SELECT dept, COUNT(*) FROM employees
WHERE salary > 30000        -- filters rows first
GROUP BY dept
HAVING COUNT(*) > 5;        -- filters groups after
Q2. Types of JOINs? HOT
  • INNER JOIN: Only matching rows in BOTH tables
  • LEFT JOIN: ALL rows from left + matching from right (NULL if no match)
  • RIGHT JOIN: ALL rows from right + matching from left
  • FULL OUTER JOIN: All rows from both tables
  • CROSS JOIN: Cartesian product (every row × every row)
  • SELF JOIN: Table joined with itself
Q3. What is a Primary Key? HOT

✅ A column (or set of columns) that uniquely identifies each row. Must be UNIQUE and NOT NULL.

Q4. Primary Key vs Foreign Key?
  • PK: Uniquely identifies a row in THIS table
  • FK: References the PK of ANOTHER table → creates relationship between tables
Q5. What is normalization? IMP

✅ Organizing data to reduce redundancy and improve integrity.

  • 1NF: No repeating groups. Each cell has single value.
  • 2NF: 1NF + no partial dependency (all non-key columns depend on full PK)
  • 3NF: 2NF + no transitive dependency (non-key depends only on PK)
Q6. DDL vs DML? HOT
  • DDL (Data Definition): CREATE, ALTER, DROP, TRUNCATE → defines structure
  • DML (Data Manipulation): SELECT, INSERT, UPDATE, DELETE → manipulates data
Q7. DELETE vs TRUNCATE vs DROP?
  • DELETE: Removes rows (with WHERE). DML. Can rollback. Slow.
  • TRUNCATE: Removes ALL rows. DDL. Cannot rollback. Fast. Resets identity.
  • DROP: Removes entire table (structure + data). DDL.
Q8. What is a subquery?

✅ A query nested inside another query.

SELECT name FROM employees
WHERE dept_id IN (SELECT id FROM departments WHERE name = 'Engineering');
Q9. What is GROUP BY?

✅ Groups rows by one or more columns for aggregate calculations.

SELECT dept, AVG(salary) FROM employees GROUP BY dept;
Q10. What is an Index? Pros and cons?
  • Pro: Speeds up SELECT/search queries dramatically
  • Con: Slows down INSERT/UPDATE/DELETE (index must be updated). Uses extra storage.
Q11. Write a query: Find the 2nd highest salary. HOT
SELECT MAX(salary) FROM employees
WHERE salary < (SELECT MAX(salary) FROM employees);

-- OR using LIMIT/OFFSET:
SELECT DISTINCT salary FROM employees
ORDER BY salary DESC LIMIT 1 OFFSET 1;
Q12. What are aggregate functions?

✅ Functions that operate on a set of rows and return a single value.

COUNT(*), SUM(col), AVG(col), MAX(col), MIN(col). NULL values are ignored (except COUNT(*)).

Q13. SQL vs NoSQL? TENNR CONFIRMED
  • SQL: Relational, structured tables, ACID, schema-based (MySQL, PostgreSQL)
  • NoSQL: Non-relational, flexible schema, horizontal scaling (MongoDB, Redis, Cassandra)
  • Use SQL for complex queries/transactions. NoSQL for scalability/unstructured data.
Q14. What is DISTINCT?

✅ Removes duplicate rows from the result set.

SELECT DISTINCT city FROM employees;
Q15. ORDER BY ASC vs DESC?

ASC: Ascending (default, A→Z, 1→9). DESC: Descending (Z→A, 9→1).

🌐 HTML / CSS — Questions (20+)
Q1. What is the CSS Box Model? HOT

✅ Every HTML element is a box: Content → Padding → Border → Margin (inside out).

box-sizing: content-box (default, width = content only) vs border-box (width includes padding + border).

Q2. What is semantic HTML? HOT

✅ Using HTML tags that convey meaning: header, nav, main, section, article, footer.

Benefits: accessibility, SEO, readability. vs non-semantic: div, span (no meaning).

Q3. Flexbox vs Grid? HOT
  • Flexbox: 1-dimensional (row OR column). Use for navbars, centering.
  • Grid: 2-dimensional (rows AND columns). Use for page layouts, dashboards.
Q4. CSS Specificity order? HOT

✅ Inline styles > ID (#) > Class (.) > Tag (element) > Universal (*)

!important overrides everything (but avoid in production code).

Q5. CSS position values?
  • static: Default. Normal flow.
  • relative: Positioned relative to its normal position.
  • absolute: Positioned relative to nearest positioned ancestor.
  • fixed: Positioned relative to viewport. Stays on scroll.
  • sticky: Toggles between relative and fixed based on scroll.
Q6. Block vs Inline vs Inline-block?
  • Block: Full width, new line (div, p, h1)
  • Inline: Only as wide as content, no width/height (span, a, strong)
  • Inline-block: Inline flow but accepts width/height
Q7. What is DOCTYPE?

✅ Declaration that tells the browser which version of HTML the page uses. <!DOCTYPE html> = HTML5.

Q8. What is z-index?

✅ Controls stacking order of positioned elements. Higher z-index = on top. Only works on positioned elements (not static).

Q9. GET vs POST? IMP
  • GET: Data in URL query string. Bookmarkable. Limited size. For reading data.
  • POST: Data in request body. Not visible in URL. No size limit. For submitting data.
Q10. What is the meta viewport tag?

✅ <meta name="viewport" content="width=device-width, initial-scale=1.0">

Makes the page responsive on mobile devices by setting viewport width to device width.

Q11. What are pseudo-classes?

✅ Select elements based on state: :hover, :focus, :active, :nth-child(), :first-child, :visited

Q12. What is media query?

✅ CSS rule that applies styles based on device characteristics (width, height, orientation).

@media (max-width: 768px) {
  .sidebar { display: none; }
}
🧮 Aptitude — Questions (15+)
A1. A is 30% more than B. B is what percent less than A?

✅ 23.08%

If B=100, A=130. Difference = 30. Percentage less = (30/130) × 100 = 23.08%

A2. A can do work in 10 days, B in 15 days. Together?

✅ 6 days

A's rate = 1/10, B's rate = 1/15. Combined = 1/10 + 1/15 = 5/30 = 1/6. Answer: 6 days.

A3. SI on Rs.5000 at 8% for 3 years?

✅ Rs.1200

SI = P×R×T/100 = 5000×8×3/100 = 1200

A4. Speed 60 km/h. Distance 180 km. Time?

✅ 3 hours

Time = Distance/Speed = 180/60 = 3 hours

A5. A:B = 3:5, B:C = 2:3. Find A:B:C?

✅ 6:10:15

Make B common: A:B = 6:10, B:C = 10:15. So A:B:C = 6:10:15

A6. CP = Rs.1500, SP = Rs.1800. Profit %?

✅ 20%

Profit = 300. Profit% = (300/1500)×100 = 20%

A7. Average of 5,10,15,20,25?

✅ 15

Sum = 75. Average = 75/5 = 15

A8. A train 200m long passes a pole in 10s. Speed?

✅ 20 m/s = 72 km/h

Speed = Distance/Time = 200/10 = 20 m/s. ×3.6 = 72 km/h

A9. What comes next: 2, 6, 12, 20, 30, ?

✅ 42

Differences: 4, 6, 8, 10, 12. Pattern: n×(n+1) → 6×7 = 42

A10. Two pipes fill tank in 3h and 6h. Together?

✅ 2 hours

Rate: 1/3 + 1/6 = 3/6 = 1/2. Time = 2 hours.

B7. Docker vs Virtual Machine? TENNR CONFIRMED
  • Docker: Lightweight containers sharing host OS kernel. Fast startup. Less isolation.
  • VM: Full OS with hypervisor. More isolation. Heavy, slow startup.
  • Containers for microservices; VMs for full OS-level isolation.
B8. Design a Rate Limiter (High Level) TENNR CONFIRMED

Purpose: Limit how many requests a user/IP can make in a time window.

  • Algorithm: Token Bucket or Sliding Window Counter
  • Storage: Redis (in-memory, fast, supports TTL)
  • Key: user_id or IP address. Value: request count + timestamp
  • Response: 429 Too Many Requests when limit exceeded
  • Scale: Use distributed Redis for multi-server setup
